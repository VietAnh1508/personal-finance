# US-WEB-008: Port Transaction Detail Edit/Delete Flow

## User Story
As a user,
I want to edit or delete existing transactions,
so that I can correct mistakes and keep records accurate.

## Acceptance Criteria
1. Transaction detail view loads income/expense, transfer, and adjustment types.
2. Edit form behavior matches transaction type-specific validation rules.
3. Editing transfer updates both linked transactions consistently.
4. Delete action confirms intent before removal.
5. On success, transactions list and totals refresh correctly.
